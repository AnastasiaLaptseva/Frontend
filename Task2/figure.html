HTML теги ‹figure› и ‹figcaption›

Описание

HTML тег ‹figure› является блочным элементом, его содержимое представляет собой самостоятельный контент, который связан с основным содержимым страницы.

Обычно он содержит:

  • картинки
  • диаграммы
  • фотографии
  • фрагменты кода
  • схемы
  • аудио
  • видео
  • рекламные объявления

Которые упоминаются в основном тексте, но которые без труда можно перенести на другую страницу или в приложение без ущерба для основного контента.

Элемент ‹figure› может содержать внутри себя элемент ‹figcaption›, который позволяет добавить подпись к контенту, он должен располагаться в качестве первого либо последнего дочернего элемента. Элемент ‹figcaption› представляет собой заголовок или легенду для ‹figure›. Не каждый тег ‹figure› нуждается в ‹figcaption›.

Только один элемент ‹figcaption› может быть помещен в ‹figure›, хотя сам элемент ‹figure›может содержать несколько дочерних элементов (например, ‹img› или ‹code›).

Пример

‹figure›
‹img src="deistvyi.jpg" alt="Мотивация"›
‹figcaption›
Мотивирующие картинки
‹/figcaption›
‹/figure›

‹figure›Элемент представляет собой некоторое содержание потока , необязательно с заголовком, который является автономным (как законченное предложение) и , как правило , ссылаются как единое целое из основного потока документа.

  • Тег figure – замкнутая единица контента. Если вы переместите элемент чуть ниже на странице или в самый низ, это никак не повлияет на его значение в документе.
  • Теги ‹figure› и ‹figcaption› являются блочными, поэтому мы можем несколько изменить форматирование нашего блока, определив дополнительные ccs-правила (Например: эффект увеличения картинки при наведении на figure)
  • Теги ‹figure› и ‹figcaption› предоставляют семантический контейнер для рисунков и чётко связывают рисунок с заголовком.
  • Изображение из тега ‹img› может быть ссылкой. Для этого нужно обернуть тег ‹img›тегом ссылки ‹a›
  • Элемент ‹img› является строчным, рекомендуется располагать его внутри блочного элемента, например, ‹p› или ‹div›.
